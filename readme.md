# Team Project Repository

## Project Overview
This repository demonstrates the use of GitHub for version control and team collaboration.

## Steps Followed

### 1. Repository Setup
A new GitHub repository was created to manage the team project.

### 2. Branch Creation
Separate branches were created for different modules of the project.

Example branches:
- main
- module1
- module2

### 3. Development in Branches
Each team member worked on their module in their respective branch.

### 4. Pull Requests
After completing the work, pull requests were created to merge branches into the main branch.

### 5. Code Review
Before merging, the code was reviewed to ensure quality and correctness.

### 6. Merge Process
Branches were merged into the main branch using GitHub Pull Requests.

### 7. Conflict Resolution
If conflicts occurred during merging, they were resolved manually.

## Tools Used
- Git
- GitHub

## Outcome
Version control was successfully implemented using GitHub with branching, pull requests, and merge conflict resolution.
